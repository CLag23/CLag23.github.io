Page Design

Home Page - Introduces the website and helps users navigate to main sections like Login, What is Python, How It’s Used, and Contact Support. It doesn’t collect data; users click buttons to access other pages.

Login Page - Lets registered users sign in using a username and password. It includes validation for required fields and redirects successful logins to the Menu page.

Menu Page - Serves as the dashboard after login, providing links to learning modules such as What is Python, How It’s Used, Set Up Environment, and Contact Support. It doesn’t require user input.

Python Page - Explains what Python is through sections like Definition, Basic Concepts, and Code Examples. Users can click links to view explanations and examples without entering any data.

How It’s Used Page - Shows real-world applications of Python in areas like Web Development, Artificial Intelligence, and Data Science. Users can click to explore each topic.

Set Up Environment Page - Guides beginners through installing Python, choosing a code editor, and using online environments. It includes external links to resources and tutorials but requires no input.

Contact Support Page - Gives users help options and includes a link to the Feedback form. It doesn’t collect data directly.

Feedback - A button that sends users to the form page

Freedback form Form Page - Collects user inquiries with fields for name, email, and message. It includes validation and buttons to submit, reset, or go back.

Dynamic Functionality

On the Login and Form pages, JavaScript will check that all fields are filled correctly before allowing users to submit. Reference: https://www.w3schools.com/js/js_validation.asp

On the Set Up Environment page, a “Next Step” button will show each setup instruction one at a time, helping users follow along. Reference: https://www.freecodecamp.org

On the How It’s Used page, clicking a topic like “Web Development” will expand or collapse additional details using JavaScript. Reference: https://www.linkedin.com
