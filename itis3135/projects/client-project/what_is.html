Language snapshot

What is Python?

A clear, readable programming language that trades noise for simplicity. Python powers teaching, startups, and large-scale systems.

Readable first

Simple syntax

Blocks are defined by indentation instead of braces. Names are clear, punctuation is minimal, and you can scan code quickly.

Batteries included

Standard library

Work with files, dates, networking, math, and testing without installing anything extra.

Flexible

Multi-paradigm

Write procedural scripts, object-oriented classes, or functional pipelines within the same project.

Typed

Dynamic with hints

Types are checked at runtime. Optional annotations improve readability and tooling without changing execution.

Core syntax at a glance

Python code is interpreted by a virtual machine. You write files, run them directly, and iterate quickly.

print("Hello, Python!")

def greet(name: str) -> str:
    return f"Hello, {name}!"

for person in ["Ada", "Linus", "Guido"]:
    print(greet(person))

Data types

Built-ins cover numbers, text, collections, and special values.

Category Examples Notes
Numbers int, float, complex Arithmetic, comparisons, and math helpers available.
Text str Immutable; supports slicing and helpful string methods.
Sequences list, tuple, range Lists are mutable; tuples are immutable.
Sets set, frozenset Unique elements; fast membership checks.
Mappings dict Key-value pairs with flexible keys.
Booleans True, False Used in comparisons and control flow.
Nothing None Represents the absence of a value.

Functions and errors

def safe_divide(a: float, b: float) -> float | None:
    try:
        return a / b
    except ZeroDivisionError:
        return None

result = safe_divide(10, 0)
print(result)

Objects

class Counter:
    def __init__(self, start: int = 0):
        self.value = start

    def increment(self) -> None:
        self.value += 1

count = Counter()
count.increment()
print(count.value)

Style and conventions

  • Follow PEP 8 for naming, spacing, and line length.
  • Write docstrings to document modules, classes, and functions.
  • Prefer explicit code, clear names, and small functions.
